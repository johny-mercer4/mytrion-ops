# Skills

Reference skills for this repo. Each `SKILL.md` is auto-surfaced by Claude Code when its
`description` matches the task, or invokable directly.

## Zoho API references (for building tool-calling integrations)

| Skill | Covers |
| --- | --- |
| [`zoho-crm-api`](zoho-crm-api/SKILL.md) | Zoho CRM REST **v8** — OAuth/scopes, metadata (modules/fields/layouts), record CRUD, search, **COQL**, related/notes/attachments/tags, **bulk read/write**, credits/rate-limits, errors. |
| [`zoho-desk-api`](zoho-desk-api/SKILL.md) | Zoho Desk **v1** — `orgId` header, metadata, tickets CRUD + actions, threads/conversations/comments, send reply, contacts/accounts, activities, search, attachments, counts, errors. |
| [`zoho-people-api`](zoho-people-api/SKILL.md) | Zoho People — the **3 coexisting API styles** (legacy forms / v2 / v3), forms/records CRUD, employees, org structure, attendance, leave, timesheets, bulk import. |

**How these fit together:**
- Auth + base URLs come from the integration wrappers: `wrapper.authHeaders('zoho_crm'|'zoho_desk'|'zoho_people')` and `zoho.baseUrl(...)` in [`src/integrations/`](../../src/integrations/) (env: `ZOHO_*` in `.env.example`).
- **Our org's live metadata** (exact module/field/form API names) is generated by `pnpm meta:zoho-crm|zoho-desk|zoho-people` → `metadataScripts/output/*.{json,md}` (git-ignored). Use the catalog for *what* exists, the skill for *how* to call.
- Build each capability as a `ToolManifest` tool dispatched via `toolDispatcher` (RBAC + department/`Administrator` gating).

> These are point-in-time research snapshots (compiled 2026-06; CRM v8, Desk v1 verified against Zoho's official OpenAPI repo). Verify against the cited official doc URLs before relying on a fine detail.
