# Octane Mini App — design spec (for Claude Design)

Telegram Mini App for **Octane**, a US fuel-card company (works with **EFS** + **WEX**). It onboards
trucking carriers: a company owner or a driver opens a one-time link inside Telegram, confirms, and
registers. A fleet owner can then hand out per-card driver links. Identity comes from Telegram
(no login form). Phone-only, lives inside Telegram.

---

## 1. Entry & identity

- Opened via a deep link: `https://t.me/octane_support_ai_bot?startapp=<registrationId>`.
- The app reads `registrationId` from Telegram `start_param` (fallback `?token=` in a browser).
- User's name / photo / language come from Telegram `initDataUnsafe.user`
  (`first_name`, `last_name`, `username`, `photo_url`, `language_code`).
- Real auth = Telegram `initData` (HMAC-verified on the backend). The app itself has no password.

## 2. Roles (3)

Derived from `profile` + `companyType`:
| Role | when |
|------|------|
| **Owner-operator** | profile = owner, companyType = owner-operator (1 fuel card, drives themself) |
| **Fleet manager** | profile = owner, companyType = fleet-manager (many cards / drivers) |
| **Driver** | profile = driver (tied to one card) |

## 3. Screens / states (state machine)

1. **Loading** — logo + spinner, "Checking your registration link…"
2. **Error** — logo (grayscale-free), title "This link isn't valid" + reason + "Contact your Octane rep…"
3. **Confirm** — the registration confirmation (owner or driver). See §5.
4. **Success** — "You're registered!"; for a **fleet manager** it continues into the **Fleet** screen (§6).
5. **Already-registered** — "Already registered" (link was used).

Signed-in screens (success / already) have a **Header** + a **Profile menu** (§4).

## 4. Header + Profile menu (signed-in — company OR driver)

- **Header** (sticky, top): brand lockup (icon + `OCTANE`) on the **left**, user **avatar on the
  right**. Tapping the avatar opens the profile menu. Safe-area padding under the notch.
- **Profile menu** = a **bottom sheet** (slides up, dimmed backdrop, drag-handle at top). Same menu
  for a company owner and a driver — only the role badge differs:
  - avatar + name + company name + **role badge** (Owner-operator / Fleet manager / Driver)
  - **Theme**: Light / Dark segmented toggle (persisted)
  - **Language**: English / Русский / O'zbek / Español (persisted; default from Telegram)
  - close by tapping the backdrop or ✕.

## 5. Confirm screen

- Brand lockup (icon + OCTANE), centered.
- Headline: "Hi {firstName}"
- Sub: owner → "Confirm you're registering:" / driver → "Confirm you drive for:"
- Detail list: **Company** = name; owner → **Account type** = Owner-operator / Fleet manager badge;
  driver → **Role** = Driver.
- Primary CTA (brand gradient): **Confirm & Register**.

## 6. Fleet screen (fleet-manager owner only)

The owner sees every active fuel card on their carrier and hands each driver a registration link.
Must scale to 20–100 cards → simple rows, NO per-row card art.

- Section header: "Your fleet" + count "{n} of {total} registered".
- **Search field** — icon + "Search card or driver"; filters by card number, driver name, network.
- **Status filter chips** (horizontal scroll): All · Registered · Invite sent · No driver — each shows
  a count; the active chip is amber-outlined.
- **Rows** — each:
  - a round **status icon** in a soft tinted circle (NOT an emoji):
    - Registered → check-badge, **green** (`--success`)
    - Invite sent (pending) → paper-plane, **brand orange** (`--primary`)
    - No driver yet (open) → user-plus, **muted**
  - masked card number, tabular: `•••• 7549` + network (`EFS` / `WEX`)
  - driver name (or "Unassigned")
  - status word on the right, in the status color
  - **open card** → expands: "Driver's name" field + **Create driver link** button
  - after create → the generated `t.me/…?startapp=…` link + **Copy**
- One card = one driver (backend-enforced). Empty search → "No cards match."

## 6a. Driver-link generation (the owner's core action)

For an unassigned card the owner: types the driver's name → taps **Create driver link** → gets a
one-time Telegram link to send that driver. Sending it flips the card to "Invite sent"; once the
driver opens it and confirms, it becomes "Registered". This is the same registration link mechanism
as §1, just generated by the owner from inside the app instead of by an admin.

## 9. Type & shape

- **Typeface: Inter Tight** (octanefuel.com's font).
- Scale (suggested): title 22/700 · eyebrow 11/600 uppercase +tracking · label 13/500 muted ·
  value 15/600 · body 14/400 · button 15/600. Card numbers = tabular figures.
- Radius: inputs/chips **10px**, buttons **12px**, cards/sheets **16px**.
- Touch targets ≥ 48px. Safe-area top & bottom. Reduced-motion respected.

## 10. Logo

Icon = black ring + transparent gap + amber→orange fuel drop (the drop nearly fills the ring, its
top edge a shallow liquid meniscus). Wordmark = `OCTANE`, Inter Tight 800, uppercase, sized ≈0.6× the
icon and set beside it. Ring color inverts with theme (black on light, white on dark); the drop's
gradient is fixed. Tagline "BEYOND FUEL" is part of the master brand mark but is NOT used in-app.

Usages:
- **Header** — small lockup (icon + OCTANE), left-aligned.
- **Loading / Confirm / Error** — larger lockup, centered.
- Icon-in-a-white-disc treatment is only for the physical fuel card (not used in the app UI now).
- Official vector: octanefuel.com. The in-app icon is currently traced SVG — swap the official
  vector when available.

## 11. Behaviours (native feel)

- Haptics on register / create-link success & failure.
- No page overscroll, no tap-highlight, no text selection except links & card numbers.
- Theme toggle also repaints Telegram's own header/background/bottom bar to match.
